The Data¶

The Enron corpus of emails and financial data has been a testing grounds for scientists for decades. It is detailed, rigorous, politically and economically relevant - it has been sifted many ways and by many different scientific disciplines. However, there are a number of characteristics of our data that leave much to be desired for the creation of a POI identification algorithm. In brief, the dataset used within contains 145 observations of 19 features as well as the 'poi' binary label. Many values are NaN's, and POI's represent only 18 of our 145 observations. With so few POI's to train our algorithm for identification, and much training variation lost to NaN's, our expectations for the performance of the algorithm ought to be somewhat modest. For the sake of providing a safe target, Udacity rubrics provide a target precision and recall of 0.3. I hope to exceed that number by as much as I can manage.

def feature_select(labels,features,alfa=0.4):
    dct = DecisionTreeClassifier(random_state=42)
    rfecv1 = RFECV(estimator=dct, step=1, cv=StratifiedKFold(labels, n_folds=6,shuffle=True,random_state=42),
                  scoring='recall')
    rfecv2 = RFECV(estimator=dct, step=1, cv=StratifiedKFold(labels, n_folds=6,shuffle=True,random_state=42),
              scoring='precision')
    rfecv1.fit(features,labels)     
    rfecv2.fit(features,labels)
    print("Optimal number of features - Recall : %d" % rfecv1.n_features_) 
    print("Optimal number of features - Precision : %d" % rfecv2.n_features_)
 
    BestFeatures = SelectFdr(score_func=f_classif,alpha=alfa)
    BestFeatures.fit_transform(features,labels)
    feature_scores = BestFeatures.scores_
    feature_pvalues = BestFeatures.pvalues_
    best_feat_indices = BestFeatures.get_support(indices=True)
    
    best_list = []
    for i in range(len(best_feat_indices)):
        best_list.append(features_list[best_feat_indices[i]+1])
        
    print 'Best features:', best_list
    feat_ctr=-1
    for index in best_feat_indices:
        feat_ctr+=1
        print best_list[feat_ctr],'Score:',feature_scores[index],'P-value:',feature_pvalues[index]
    
    plt.figure()
    plt.xlabel("Number of features selected")
    plt.ylabel("Recall & Precision")
    plt.plot(range(1, len(rfecv1.grid_scores_) + 1), rfecv1.grid_scores_,
             label='Recall',color='blue')
    plt.plot(range(1, len(rfecv2.grid_scores_) + 1), rfecv2.grid_scores_,

Feature Analysis¶

Because the dataset has very few POI's, general accuracy does not provide the best measure for a feature's power in identifying POI's - if a feature identified every point as non-POI, for instance, it would still attain an accuracy score of nearly 88%. As such, we will use Recall and Precision as our means of judging the efficacy of our features.

For the same reason, I chose to use the SelectFdr method within the feature_select function. SelectFdr allows the user to adjust selection based on the feature's false discovery rate rather than its accuracy. False discovery rate is tied directly to precision and recall rather than the identification of 'true negatives' alone, making it a better method than SelectKBest or SelectPercentile, which primarily use accuracy to rate features.

It's plain to see that our selected 'best features' have very poor p-values and low accuracy scores even with an alpha of 0.4. Rather than depend on our recursive feature selection now, I chose to hand-pick three variables that, based on knowledge of the Enron case, would seem to correlate well with POI identification. Using these features, I can analyze any issues with our data.

Classifier Testing¶

Now that we have strong features to use in identifying our POIs, we will create and fit some classifiers. I fit different classifiers as well as a 'soft' voting classifier, which takes the probabilities of the previous four fits and uses them to make a combined prediction.

As StratifiedShuffleSplit() is used in the grading program, I have used that same method for cross-validation. Stratified CVs are especially useful in the case where targets are relatively rare, as our POIs are. A tally is created for the best classifier (identified by F1 score) of true/false positives/negatives in order to produce precision and recall scores.

Concluding Q&A¶

Summarize the goal of this project - give some background on the dataset and how it can be used to answer the project question. Were there any outliers in the data when you got it, and how did you handle those?¶

This project involved using a variety of financial and email information to identify relatively rare individuals within our dataset (people of interest). I used supervised machine learning algorithms to train classifiers of a few different varieties. These algorithms were trained on features selected with a recursive feature elimination algorithm as well as hand-picking features based on qualititive knowledge of the Enron case.

There was one consistent outlier across many feature categories - the 'TOTAL' dictionary entry. It was easily identified once the data were visualized in boxplots. I eliminated the 'TOTAL' entry from the dataset, then checked each dictionary entry to ensure that no other totaling entry existed.

There were still outliers in the technical sense after this process - however, they were often the targets of our algorithm and therefore desireable to keep.

What features did you end up using in your identifier, and what selection process did you use to pick them? Did you have to do any scaling? Why or why not?¶

In your feature selection, if you used an algorithm like a decision tree, please also give the feature importances of the features that you use, and if you used an automated feature selection function like SelectKBest, please report the feature scores and reasons for your choice of parameter values.¶

Based on their accuracies and p-values, reported as attributes from the sklearn RFECV function, I chose the following six features for my feature list:

  1. Salary
  2. Bonus
  3. Deferred Income
  4. Total Stock Value
  5. Exercised Stock Options
  6. Long Term Incentive

Their accuracy scores and p-values are reported above.

I had considered doing principal component analysis to squeeze some extra value from the discarded features - however, I believe that these six variables act well enough as approximations for their 'classes' of feature, and rather than discard the information held in the noise of these six features, as would be necessary for PCA, I chose to keep them intact.

I did combine the POI-related email features into a single variable, simply adding their entries together. The new variable did not prove to be much stronger than the strongest of the features used to create it.

What algorithm did you end up using? What others did you try? How did model performance differ between algorithms?¶

The sklearn Gaussian Naive Bayes classifier ended up being my strongest classifier for this problem. I also fit a gradient boosting algorithm, an AdaBoost algorithm (decision tree base estimator), a random forest classifier, an SVC with 'rbf' kernel, and a 'soft' voting classifier that incorporated probability information from each to categorize points.

To my surprise, GaussianNB() outperformed each other classifier significantly. The F1 scores for each classifier are reported in the body of this document, with GaussianNB() having nearly double the F1 score of each other classifier.

What does it mean to tune the parameters of an algorithm, and what can happen if you don’t do this well? How did you tune the parameters of your particular algorithm?¶

Algorithm tuning parameters allow the programmer to optimize the algorithm for his or her needs. For example, support vector machines use a 'penalty' parameter, 'C', that affects how closely fit the classifier will be to the data.

Parameters fundamentally change the shape of the decision space of the classifier, but ultimately, they are subject to bias-variance tradeoff. If your classifier fits your training data too well and cannot generalize, you have 'overfit' the classifier. This is where the finesse of parameter tuning happens.

I had originally implemented a GridSearchCV parameter search to try to optimize my parameters, but even after quite a bit of work, the default GaussianNB() remained the strongest algorithm.

What is validation, and what’s a classic mistake you can make if you do it wrong? How did you validate your analysis?¶

Cross-validation is a primary means of protecting yourself from overfitting your classifier or biasing it incorrectly based on an unlikely random assortment of the points. Cross-validation (in this case, StratifiedShuffleSplit) runs the classifier a number of times on the same dataset, but uses different configurations of the data to protect against using a classifier that doesn't actually work well with the data.

A 'stratified' cross-validation method is used in this study because there are relatively few POI observations - the stratification keeps the proportion of non-POI and POI observations consistent between validation interations. If stratified validation is not used, many of the validations will be performed without POI observations at all.

There is some risk in conducting feature selection before cross-validation as I have - it will bias the model since testing data were used in the model's creation. However, the alternative would be to combine feature selection with cross-validation, which would entail gathering information about feature importances from each iteration of the cross-validation, significantly complicating the process.

Give at least 2 evaluation metrics and your average performance for each of them. Explain an interpretation of your metrics that says something human-understandable about your algorithm’s performance¶

Our classifiers must be rated based on how often they correctly classify data. As our classification problem for this project is binary (POI or not), there are only four possible classification outcomes, two good and two bad:

  1. True Positive (Identified as POI, Is POI)
  2. True Negative (Identified as non-POI, Is non-POI)
  3. False Positive (Identified as POI, Is Not POI)
  4. False Negative (Identified as non-POI, Is Not non-POI)

True Positives and Negatives are what we seek to maximize - correct identifications of POIs and non-POIs. Combinations of these values allow us some insight into the success of our classifiers. Precision, for example, is the proportion of true positives to the sum of true and false positives - precision is a measure of how well our algorithm can tell the difference between POI and non-POI. Recall, on the other hand, is a measure of how sensitive our algorithm is to finding POIs at all, which is to say, out of all POI's, what proportion were found as true positives. The composition of these two metrics gives the classifier's F1 score. Our best classifier's scores are as follows:
